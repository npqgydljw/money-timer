## 一、标准格式

每个字段的描述必须遵循以下标准格式：

```
字段名称：字段类型，必填性，字段限制，默认值，其他说明（可选）
```

**五要素说明**：
1. **字段名称**：清晰的业务名称
2. **字段类型**：输入控件类型
3. **必填性**：必填/选填
4. **字段限制**：长度、格式、取值范围
5. **默认值**：初始值或默认选项

## 三、字段属性完整说明

### 3.1 必填性
- **必填**：用户必须填写才能提交，通常用红色星号（*）标识
- **选填**：用户可以选择性填写

### 3.2 字段限制

#### 长度限制
- **限X字内**：限制输入字符数量
- **限X字符内**：同上
- **最小X字**：最少输入X个字符
- **X-Y字**：长度范围

示例：
```
用户名：输入框，必填，限20字内。
备注：文本域，选填，限500字内。
```

#### 格式限制
- **仅数字**：只能输入数字（0-9）
- **仅字母**：只能输入字母（a-z、A-Z）
- **数字和字母**：只能输入数字和字母
- **手机号格式**：11位数字，以1开头
- **邮箱格式**：符合邮箱标准格式
- **URL格式**：符合URL标准格式

示例：
```
手机号：输入框，必填，11位数字，以1开头。
邮箱：输入框，必填，符合邮箱标准格式。
```

#### 取值范围
- **最小值X**：数值的最小值
- **最大值Y**：数值的最大值
- **X-Y之间**：数值范围

示例：
```
年龄：数字输入框，必填，18-100岁之间。
数量：数字输入框，选填，最小值1，最大值9999。
```

#### 选项列表（下拉类型）
- **下拉选项包括A、B、C**：列出所有选项
- **单选/多选**：说明是单选还是多选
- **可清空/不可清空**：说明是否可以清除选择

示例：
```
性别：下拉选项包括男、女、保密。默认选择保密。单选。可清空。
标签：下拉选项包括标签1、标签2、标签3。默认不选。多选。可清空。
```

### 3.3 默认值
- **默认为空**：初始状态为空
- **默认选中"XX"**：下拉或单选的默认选项
- **默认不选**：下拉或多选的初始状态
- **自动填充**：系统自动填充（如当前时间、当前用户）
- **默认值为X**：具体的默认值

示例：
```
状态：下拉选项包括启用、禁用。默认选中"启用"。单选。
创建时间：日期选择器，选填，默认为空。
创建人：输入框，必填，自动填充为当前登录用户。
```

### 3.4 其他说明（可选）
- **只读**：字段不可编辑，仅展示
- **禁用**：字段不可用
- **隐藏**：字段不显示
- **唯一性**：字段值必须唯一
- **关联查询**：需要从其他数据源查询
- **特殊格式**：如日期格式 YYYY-MM-DD

示例：
```
创建时间：输入框，选填，只读，格式为YYYY-MM-DD HH:mm:ss。
用户ID：输入框，选填，唯一性。
```

## 四、常见字段描述示例

### 4.1 用户相关字段
```
用户名：输入框，必填，限20字内，仅字母和数字，唯一性。
昵称：输入框，选填，限30字内。
真实姓名：输入框，必填，限10字内，仅中文。
手机号：输入框，必填，11位数字，以1开头，唯一性。
邮箱：输入框，选填，限50字内，符合邮箱标准格式，唯一性。
密码：密码输入框，必填，限20字内，必须包含字母和数字。
头像：图片上传，选填，支持jpg/png格式，大小不超过2MB。
```

### 4.2 商品相关字段
```
商品名称：输入框，必填，限100字内。
商品编码：输入框，必填，限50字内，唯一性。
商品分类：级联下拉（一级分类-二级分类），必填，默认不选。
商品价格：数字输入框，必填，最小值0.01，保留2位小数。
库存数量：数字输入框，必填，最小值0，最大值99999。
商品描述：富文本编辑器，选填，限5000字内。
商品图片：图片上传，必填，支持jpg/png/jpeg格式，最多上传9张，每张大小不超过5MB。
商品标签：标签输入，选填，最多选择5个。
上架状态：下拉选项包括上架、下架。默认选中"下架"。单选。
```

### 4.3 订单相关字段
```
订单号：输入框，选填，只读，系统自动生成，唯一性。
下单时间：日期时间选择器，选填，只读，系统自动填充。
订单金额：数字输入框，选填，只读，保留2位小数。
收货人：输入框，必填，限20字内。
联系电话：输入框，必填，11位数字。
收货地址：级联下拉（省-市-区）+ 详细地址输入框，必填。
订单状态：下拉选项包括待支付、已支付、已发货、已完成、已取消。单选，不可清空。
支付方式：下拉选项包括微信支付、支付宝、银行卡。单选。
```

### 4.4 系统管理字段
```
菜单名称：输入框，必填，限30字内。
菜单图标：下拉选择图标，选填。
父菜单：级联下拉（一级菜单-二级菜单），选填，默认不选。
菜单路径：输入框，必填，限100字内。
排序号：数字输入框，选填，最小值0，默认为0。
启用状态：下拉选项包括启用、禁用。默认选中"启用"。单选。
创建时间：日期时间选择器，选填，只读，格式为YYYY-MM-DD HH:mm:ss。
```

### 4.5 查询条件字段
```
关键词：搜索输入框，选填，支持模糊搜索，可清空。
时间范围：日期范围选择器，选填，默认不选，可清空。
状态：下拉选项包括全部、启用、禁用。默认选择全部。单选。可清空。
类型：下拉选项包括类型1、类型2、类型3。默认选择全部。单选。可清空。
创建人：输入框，选填，支持模糊搜索，可清空。
```

## 五、注意事项

### 5.1 必须遵循的原则
1. **完整性**：五要素必须全部描述清楚
2. **准确性**：限制条件和默认值必须准确
3. **一致性**：同类字段的描述方式要保持一致
4. **明确性**：避免使用"请输入"等模糊描述

### 5.2 常见错误
❌ 错误示例：
```
用户名：请输入用户名。
```
- 问题：缺少字段类型、必填性、限制、默认值

✅ 正确示例：
```
用户名：输入框，必填，限20字内，仅字母和数字，唯一性。
```

❌ 错误示例：
```
商品价格：数字，必填。
```
- 问题：缺少取值范围（最小值、小数位数）

✅ 正确示例：
```
商品价格：数字输入框，必填，最小值0.01，保留2位小数。
```

### 5.3 特殊情况处理
1. **联动字段**：需要说明字段之间的联动关系
   ```
   省份：级联下拉（省），必填，默认不选。
   城市：级联下拉（省-市），必填，默认不选，根据省份动态加载。
   区县：级联下拉（省-市-区），选填，默认不选，根据城市动态加载。
   ```

2. **组合字段**：多个字段组合使用
   ```
   收货地址：级联下拉（省-市-区）+ 详细地址输入框，必填。
   详细地址：输入框，必填，限200字内。
   ```

3. **动态选项**：选项从接口获取
   ```
   部门：下拉选项，必填，默认不选，选项从组织架构接口获取。
   ```

### 5.4 字段验证提示
在 PRD 中，除了字段描述，还应说明验证失败时的提示信息：

```
手机号：输入框，必填，11位数字，以1开头，唯一性。
验证规则：
- 如果为空，提示"请输入手机号"
- 如果格式不正确，提示"请输入正确的手机号"
- 如果已存在，提示"该手机号已被注册"
```
