# PRD 标准结构模板

## 目录
- 一、整体文档结构
- 二、功能模块的 5 部分结构
- 三、交互说明的细化要求
- 四、字段描述标准格式
- 五、业务逻辑描述方式

## 一、整体文档结构

PRD 文档应采用模块化分层结构，包含以下主要部分：

```
PRD 文档
├── 产品概述
│   ├── 产品名称
│   ├── 产品定位
│   ├── 目标用户
│   └── 核心价值
├── 产品功能详细设计
│   ├── 公共模块（跨功能共享）
│   │   ├── 租户管理
│   │   ├── 权限管理
│   │   └── 字典管理
│   └── 具体功能模块
│       ├── 用户管理
│       ├── 订单管理
│       └── 统计分析
└── 附录
    ├── 术语说明
    └── 参考文档
```

## 二、功能模块的 5 部分结构

每个功能模块必须包含以下 5 个部分，顺序固定：

### （1）需求描述
**目的**：说明功能的目的、业务场景和价值

**内容要求**：
- 清晰描述功能要解决的问题
- 说明业务场景和使用流程
- 阐述功能带来的价值（提升效率、降低成本、改善体验等）

**示例**：
```
（1）需求描述
各省都是一个子租户，在子租户之上有一个主租户（比如集团客服部），
可以看到各个子租户的信息。租户新增管理范围。租户下的组织的管理范围
跟随租户。实现多租户机制，确保同一个应用资源可以分配给多个租户使用。
```

### （2）前置条件
**目的**：使用该功能需要满足的先决条件

**内容要求**：
- 列出使用该功能前必须满足的条件
- 可以是系统配置、权限、数据准备等
- 如无特殊条件，标注"无"

**示例**：
```
（2）前置条件
无。
```

或

```
（2）前置条件
1. 用户已登录系统
2. 用户具有"租户管理"权限
3. 已完成字典配置
```

### （3）交互说明
**目的**：详细描述 UI 布局、操作流程、字段定义（核心部分）

**内容要求**：详见"三、交互说明的细化要求"

### （4）后置条件
**目的**：功能执行后的状态变化或系统影响

**内容要求**：
- 说明操作成功后的系统状态
- 说明对其他模块的影响
- 如无特殊影响，标注"无"

**示例**：
```
（4）后置条件
租户创建成功后，系统自动为该租户初始化基础数据，包括默认角色、默认权限等。
```

或

```
（4）后置条件
无
```

### （5）异常说明
**目的**：错误处理和异常情况说明

**内容要求**：
- 列出可能出现的异常情况
- 说明异常的处理方式（提示信息、跳转流程等）
- 如无异常处理，标注"无"

**示例**：
```
（5）异常说明
1. 如果有被引用，则提示"是否删除 xxxx"。xxxx为租户名称。
2. 删除失败时，显示错误信息："删除失败，请稍后重试"。
```

## 三、交互说明的细化要求

交互说明是 PRD 的核心部分，必须详细描述以下内容：

### 3.1 查询条件
**要求**：列出所有筛选字段，说明字段属性

**必填信息**：
- 字段名称
- 字段类型（输入框、下拉、日期等）
- 默认值
- 是否可清空
- 单选/多选（针对下拉）
- 模糊/精确（针对输入框）

**示例**：
```
查询条件
1、租户名称：输入框支持模糊搜索，可清空框内数据。
2、租户类型：下拉选项包括全部、子租户、主租户。默认选择全部。单选。无法清空。
3、创建时间：日期范围选择器，默认不选。可清空。
```

### 3.2 列表字段展示
**要求**：明确展示字段和排序规则

**必填信息**：
- 排序规则（默认排序字段和排序方式）
- 展示字段列表
- 分页方式（如有）

**示例**：
```
列表字段展示
1、列表默认排序规则：以创建时间降序排列。
2、列表展示字段：租户名称、租户类型、主管理员、租户人数、租户组织树、状态、创建时间。
3、分页方式：每页显示 20 条，支持跳转页码。
```

### 3.3 操作权限
**要求**：详细说明每个操作的交互逻辑

**必填信息**：
- 操作名称（如新增、编辑、删除）
- 触发方式（按钮点击、右键菜单等）
- 交互流程（弹窗、跳转页面、确认提示）
- 权限控制（哪些角色可见）

**示例**：
```
【新增】
点击【新增租户】按钮，中央弹出带遮罩弹窗。默认选中"子租户"标签页。

【详情】
点击按钮，中央弹出带遮罩弹窗。
一个租户只能有一个主管理员。主管理员只能转移。

【删除】
如果被引用，则提示"是否删除 xxxx"。xxxx为租户名称。
```

### 3.4 表单字段定义
**要求**：每个表单字段必须包含完整属性

**标准格式**：
```
字段名称：字段类型，必填性，字段限制，默认值
```

**必填信息**：
- 字段名称
- 字段类型（输入框、下拉、日期、级联、文本域等）
- 必填性（必填/选填）
- 字段限制（长度、格式、取值范围）
- 默认值

**示例**：
```
租户名称：输入框，必填，限50字内。
主管理员：输入框，必填，限10字内。
租户性质：下拉选项目前只有单位，单选，选填，默认选中"单位"。
租户所属区域：下拉选项：行政区域（仅包含省）级联下拉。单选。默认不选。不能清空。
```

### 3.5 业务逻辑描述
**要求**：对复杂流程使用条件分支清晰描述

**描述方式**：
- 使用"如果...则..."的句式
- 多层判断使用缩进或编号
- 说明每种情况的返回结果

**示例**：
```
如果输入的短信验证码正确，则判断账号是否在管理员设置的有效期内，
如果不在账号有效期内，吐司提示："该账号不在有效期内，请联系管理员"。

如果处于账号有效期内，则判断账号状态：
如果账号状态为"密码过期"，则吐司提示："您的密码已过期，请修改密码"。
如果账号状态为"注销"，则吐司提示："该账号处于注销状态，请联系管理员"。
```

## 四、字段描述标准格式

### 4.1 字段类型说明
- **输入框**：单行文本输入
- **文本域**：多行文本输入
- **下拉**：从固定选项中选择
- **级联下拉**：多级关联下拉（如省-市-区）
- **日期选择器**：选择日期或日期范围
- **数字输入框**：仅允许输入数字
- **复选框**：多选
- **单选框**：单选
- **文件上传**：上传文件

### 4.2 字段限制说明
- **长度限制**：限X字内、限X字符内
- **格式限制**：仅数字、仅字母、邮箱格式、手机号格式
- **取值范围**：最小值X、最大值Y
- **选项列表**：下拉选项包括A、B、C

### 4.3 必填性说明
- **必填**：用户必须填写才能提交
- **选填**：用户可以选择性填写

### 4.4 默认值说明
- 明确默认值是什么
- 说明默认值是自动填充还是用户选择
- 如无默认值，说明"默认不选"或"默认为空"

## 五、业务逻辑描述方式

### 5.1 条件分支描述
使用"如果...则..."句式，清晰描述业务逻辑：

```
如果条件A满足，则执行操作A。
如果条件A不满足且条件B满足，则执行操作B。
如果条件A和B都不满足，则执行操作C。
```

### 5.2 多层判断描述
使用编号或缩进表示层级关系：

```
1. 判断输入的手机号码是否是平台账号
   如果是平台账号，则下发短信验证码
   如果不是，则在手机号码输入框下，使用红色字体提示："请输入正确的手机号码"

2. 判断短信验证码是否在时效范围内
   如果不在时效范围内，则在验证码输入框下，使用红色字体提示："您的验证码已失效，请重新获取"
   如果在时效范围内，判断输入的短信验证码是否正确
     如果不正确，则在验证码输入框下，使用红色字体提示："请输入正确的短信验证码"
```

### 5.3 提示信息描述
- 明确提示信息的出现位置（吐司、弹窗、字段下方红色字体）
- 引用提示信息内容时使用双引号
- 说明提示信息的触发条件

### 5.4 权限控制描述
- 明确每个操作的角色权限
- 说明不同角色看到的界面差异

**示例**：
```
该页面由超级管理员和主租户管理员来管理操作。
超级管理员可以查看管理主租户和子租户，以及主子租户之间的关系管理。
主租户管理员可以管理查看本身和其主租户下所有创建的子租户。
```
